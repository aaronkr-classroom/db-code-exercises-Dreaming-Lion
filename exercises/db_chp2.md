## 연습문제

1. (______) 안에 알맞은 용어는?

2. (______) 안에 알맞은 용어는?
개넘 세계의 데이터는 개체의 의미로부터 인식된 개념으로， 둘 이상의 속성으로 구성된 (______)으로표현된다. >>> 개체 타입

3. 현실 세계의 직장인이 속한 세계에 따라 구분되는 구성 요소를 5개 이상 작성하시오.

| 구분 | 구성요소 |
|-----|----------|
| 현실 세계의 직장인 | 이름, 나이, 성별, 키, 취미 |
| 회사라는 개념 세계의 직장인 | 사번, 부서, 직급, 급여, 입사일, 담당 업무 |
| 자동차 보험사라는 개념 세계의 직장인 | 사번, 부서, 직급, 급여, 입사일, 담당 업무 |
| 회사라는 개념 세계와 대응하는 컴퓨터 세계의 직장인 | 사번 데이터, 이름 데이터, 부서 코드, 직급 코드, 급여 값, 입사일 값, 연락처 데이터 |

4. 데이터 모델령의 각단계와설명을바르게 연결하시오. >>> (1 - c), (2, a), (3 - b)
  1. 개념적스카마모텔링
  2. 논리적스키마모델링
  3. 물리적스키마모델렁
    a. 개념적 모델링 결과인 개념적 구조를 DBMS가이해할 수 있는 논리적 구조， 즉 데이터 모델로 변환하는과정
    b. 디스크에 데이터가 저장될 수 있도록 논리적 구조를 물리적 구조로 변환하는 과정
    c. 현실 세계에서 인석한 데이터를 추상화를 통해서 개념 세계의 추상적 개념으로 표현하는 과정

5. 데이터 모텔링을 개념적 모댈링， 논리적 모댈링， 물리적 모댈링으로 구분해서 수행히는 이유를 설명하시오.
>>> 데이터 모델링을 개념적, 논리적, 물리적 모델링으로 구분하는 이유는 사용자 관점, 논리적 구조 관점, 물리적 저장 관점을 분리하여 단계적으로 데이터베이스를 설계하고, 논리적 데이터 독립성과 물리적 데이터 독립성을 확보하기 위해서이다. 또한 현실 세계의 복잡한 데이터를 추상화하여 이해하기 쉽게 표현하고, 이를 구현 가능한 구조로 체계적으로 변환할 수 있다.



6. 개념적 구조， 즉 개념적 모댈링 결과를 DBMS가 직접 이해할수 없는 이유를 설명하시오.
>>> 개념적 모델링 결과는 현실 세계를 사용자 중심으로 추상화한 것으로, 특정 DBMS에 종속되지 않은 표현이기 때문에 DBMS가 직접 이해할 수 없다. 따라서 이를 DBMS가 처리할 수 있는 논리적 데이터 구조로 변환해야 한다.
