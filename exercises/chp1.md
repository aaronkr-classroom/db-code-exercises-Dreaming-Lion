9. 3단계 DB 구조가 어떻게 데이터 독립성을 제공하는지 설명하시오
    >>> 3단계 DB 구조는 외부 스키마, 개념 스키마, 내부 스키마로 데이터베이스를 나누어 바라보는 방식이며, 이 구조의 가장 큰 목적은 데이터 독립성을 확보하는 데 있다. 데이터 독립성이란 한 단계에서 변화가 일어나더라도 그 변화가 다른 단계에 최소한으로만 영향을 주도록 만드는 성질을 의미한다. 즉, 데이터 저장 방식이 바뀌거나 사용자 관점이 바뀌어도 전체 시스템을 전부 수정하지 않고 유지할 수 있게 해 주는 것이다.
    >>> 먼저 외부 스키마는 사용자나 응용 프로그램이 보는 데이터의 모습이다. 예를 들어 학생 관리 시스템에서는 교직원은 학번, 이름, 성적을 볼 수 있고, 도서관 시스템은 학번, 이름, 대출 정보만 볼 수 있다. 이렇게 사용자마다 필요한 데이터만 다르게 볼 수 있도록 한 것이 외부 스키마이다. 그 아래의 개념 스키마는 데이터베이스 전체의 논리적 구조를 정의한다. 여기서는 학생, 과목, 수강, 성적 같은 데이터가 어떤 관계를 가지는지 전체적인 설계를 표현한다. 마지막으로 내부 스키마는 데이터가 실제로 디스크에 어떻게 저장되는지를 나타낸다. 파일 구조, 인덱스, 저장 위치, 압축 방식 등이 여기에 해당한다.
    >>> 이 3단계 구조가 데이터 독립성을 제공하는 이유는 각 단계가 서로 분리되어 매핑되어 있기 때문이다. 예를 들어 내부 스키마에서 저장 구조를 바꾸어 인덱스를 추가하거나 파일 저장 방식을 변경하더라도, 개념 스키마 자체는 그대로 유지할 수 있다. 그러면 응용 프로그램이나 사용자는 이런 물리적 변화를 거의 느끼지 못한다. 이것을 물리적 데이터 독립성이라고 한다. 즉, 데이터의 실제 저장 방식이 바뀌어도 논리적 구조나 사용자 화면은 영향을 적게 받는다.
    >>> 또한, 개념 스키마가 바뀌더라도 외부 스키마를 적절히 유지하면 사용자 입장에서는 큰 변화 없이 계속 데이터를 사용할 수 있다. 예를 들어 원래 학생 테이블에 있던 주소를 별도의 주소 테이블로 분리했다고 하더라도, 사용자에게는 여전히 학생 정보와 주소가 함께 보이도록 외부 스키마를 조정할 수 있다. 이처럼 논리적 구조의 변경이 사용자별 뷰에 직접적인 혼란을 주지 않도록 하는 것을 논리적 데이터 독립성이라고 한다.
    >>> 결국 3단계 DB 구조는 데이터베이스를 사용자 관점, 전체 논리 관점, 물리 저장 관점으로 나누어 관리함으로써, 어느 한 부분이 바뀌어도 다른 부분에 미치는 영향을 줄여준다.
